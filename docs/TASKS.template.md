# Tasks / 作業リスト

> `/tasks` コマンドが `docs/SPEC.md` から自動生成します。
> 非エンジニアの方も読めるよう、各タスクは「**何ができるようになるか**」を平易な日本語で書きます。
> 技術的な詳細は `<details>` で折りたたんであるので、AIが参照する用です（人間はクリックして開く必要なし）。

## 進捗の見方
- `[ ]` まだ未着手
- `[~]` AIが作業中
- `[x]` 完成・視覚チェック合格
- `[!]` 詰まったので人間に相談したい

---

## タスク一覧

### [ ] T1: <ひとことで何を作るか>
**完成すると何が見えるか / What the user will see when done**
- 例: トップページに「ログイン」ボタンが表示される
- 例: ボタンを押すとログイン画面に移動する

**完成の目安 / Done when**
- [ ] 上記の見た目になっている（AIがブラウザのスクリーンショットで確認）
- [ ] エラーが出ていない

<details>
<summary>🔧 技術的な詳細（AI用）/ Technical details (for AI)</summary>

- SPEC ref: §
- Files to touch:
  - `src/...`
- Steps:
  1.
- Quality gates: typecheck, lint, no console errors
</details>

---

## 進め方
1. 上から順に AI が1つずつ実装します
2. 各タスクが終わるごとに AI 自身がブラウザで確認します
3. 全部 `[x]` になったら完成
4. `[!]` がついたら人間の助けが必要な合図です
